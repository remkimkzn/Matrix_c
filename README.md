## Проект: Работа с матрицами на языке C

**Описание:**  
Реализация библиотеки для работы с матрицами на языке C. Все основные операции с матрицами реализованы с учётом стандартов и принципов структурного программирования. Программа поддерживает создание, очистку, сравнение, сложение, вычитание, умножение, транспонирование, вычисление миноров, определителей и обратных матриц.

**Технологии и инструменты:**  
- Язык: C (C11)
- Операционная система: Debian 12 (GNU/Linux)
- Сборка: `Makefile`
- Библиотека: Статическая библиотека с заголовочным файлом `s21_matrix.h`
- Тестирование: Unit-тесты с использованием библиотеки `Check`
- Стандарты: POSIX.1-2017, Google Style

### Операции над матрицами

Все операции (кроме сравнения матриц) возвращают результирующий код:

| Код | Описание                          |
|-----|-----------------------------------|
| 0   | OK                                |
| 1   | Ошибка, некорректная матрица      |
| 2   | Ошибка вычисления (несовпадающие размеры и т.д.) |

#### Основные операции:
- **create_matrix** — создание матрицы
- **remove_matrix** — очистка матрицы
- **eq_matrix** — сравнение матриц
- **sum_matrix** — сложение матриц
- **sub_matrix** — вычитание матриц
- **mult_number** — умножение матрицы на число
- **mult_matrix** — умножение двух матриц
- **calc_complements** — вычисление минора и алгебраических дополнений
- **determinant** — вычисление определителя матрицы
- **inverse_matrix** — нахождение обратной матрицы

### Реализация функций библиотеки

Библиотека должна быть реализована в виде статической библиотеки с заголовочным файлом `s21_matrix.h`. Перед каждой функцией используется префикс `s21_`. Примеры операций:
- **create_matrix**: создание матрицы с заданными размерами
- **remove_matrix**: очистка и освобождение памяти
- **eq_matrix**: сравнение двух матриц на равенство
- **sum_matrix**: сложение двух матриц (с проверкой корректности размеров)
- **mult_matrix**: умножение матриц с учётом правил линейной алгебры

### Требования к реализации
- Использование компилятора gcc с поддержкой стандарта C11
- Соблюдение стандартов POSIX.1-2017, отказ от устаревших функций
- Придерживание стиля кодирования Google Style
- Покрытие всех функций unit-тестами (не менее 80% покрытия каждой функции)
- Для тестирования используется библиотека `Check`
- Создание отчёта с использованием gcov для тестов с флагом gcov

### Сборка проекта

Для сборки библиотеки и тестов используется `Makefile` с целями:
- **all** — сборка библиотеки
- **clean** — очистка сборочных файлов
- **test** — запуск unit-тестов
- **s21_matrix.a** — создание статической библиотеки
- **gcov_report** — создание отчёта gcov в формате HTML

Пример команд для сборки:
```bash
make all
make test
make gcov_report
